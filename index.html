Overview

Dishwasher is a Kitchen Sink bot for Discord, originally made for the OneShot Discord Server . Refrigerator is its Revolt counterpart. Both provide moderation capabilities, alongside quality of life features to make moderating a server less intensive.

For the sake of this documentation, the bots will be referred to as Appliances.

Please note

Dishwasher is the primary project, while Refrigerator is currently in development. Some features from Dishwasher may be missing in Refrigerator.

Features

Here is a non-exhaustive list of the notable Appliance features.

  • User reflection commands (info, avy, joinscore…)
  • Translation of messages using DeepL.
  • Quoting of linked messages within servers that the Appliance is in.
  • Custom prefixes per-user.
  • Reminders in DMs.
  • Per-user timezone commands.
  • Polling.
  • A “Color of The Day” system for a server’s role.
  • User configurable configs.
  • Staff moderation logs.
  • Channel locking.
  • User watching.
  • User “tossing” system.
  • Raidmode.
  • Puppeteering.

All features listed here work or mostly work.

Inviting

At the moment, both Dishwasher and Refrigerator are not public bots. Instead, they are “semi-public”.

The former is due to Discord’s 100 server limit for non-verified bots. Verification is infeasible due to the construction of the bot.

The latter is due to the bot being in development.

However, you are welcome to use either bot should you know renavi. Simply ask.

Advanced Functionality

Commands that have a more specific use, or are part of a specific feature set, are put in this section.

Some features may require setup beforehand. Please see the configuration sections.

Non-specific

These aren’t specific to any feature set, and can be used freely.

Information

  • info [user (yourself)]
    Gets information on a user.

  • info server [server (current)]
    Gets information on a server.

  • info role [role ID or name (@everyone)] Gets information on a role.

Join*

  • joinscore {joinscore or mention}
    Posts your (or someone else’s) joinscore for the current server.

  • joingraph
    Posts a graph of times people have joined.

Reminders

  • remind [time] [text]
    Reminds you about something in Direct Messages.
    Use “number then unit” format, such as 5s, which is the minimum.

    Lists your reminders.

    Removes a reminder using the index from the reminders command.

    Shows the timezone for you, or someone else.

Messagescan

“Messagescan” is the name for the external site linking feature set, that currently supports Discord, Twitter/X, and DeepL.

Use of this feature set requires embed_enable and translate_enable set to True in the Misc page of the server configuration.

If you paste a Discord message link, the Appliance will quote the message, with a pretty embed.

If you paste a Twitter/X link, the Appliance will send a vxtwitter.com link in its stead, suppressing the original embed for you (if it has MANAGE_MESSAGES).

User Prefixes

By default, each user can have a maximum of 3 custom prefixes. This can be adjusted in the Appliance’s configuration. Be warned that the maximum is 24.

  • prefixes
    Shows your currently active prefixes.

  • prefixes add [prefix]
    Adds a new prefix.

  • prefixes remove [index]
    Removes a prefix using the index from the prefixes command.

Color of The Day

Keep in mind

This feature is currently unimplemented in Refrigerator.

Color of The Day (or “CoTD”) is a feature that will change a role’s color randomly each day.

To use, the Cotd page in the server’s configuration must be filled out, and enable must be set to True.

Please note

At the moment, only one role can be set as the CoTD role.

User Commands

  • cotd
    Displays the current Color of The Day.

  • voteskip
    Votes to skip the current Color of The Day by moving the roll time an hour earlier.

Staff Commands

  • reroll
    Forcibly rerolls the current Color of The Day.

Moderation Functionality

Up until now, most commands have been solely ones that users could use (save for the administrative reroll command). Most of the commands in this section are Staff only (as you can guess).

Before proceeding, it’s important to know what constitues as “Staff”. The Appliances will recognize any of the following as a Staff member:

  • A user with the configured staff_role.
  • Any of the bot’s managers.
  • A user with the MANAGE_GUILD permission.

Non-specific

These aren’t specific to any feature set, and can be used freely.

Staff

  • ban [target] {reason}
    Bans a user and messages them with the reason.

  • dban [target] [duration] {reason}
    Bans a user with a specified amount of days worth of messages deleted, messages them with the reason.

  • kick [target] {reason}
    Kicks a user.

  • lock {channel (current)} {soft (false)}
    Prevents people from speaking. Will have a “hard” tone by default.
    Use true/false with the “soft” argument to adjust this behavior.

  • massban [targets]
    Bans several users with their IDs without messaging them.

  • modtoggle
    If exstaff_role is configured, toggles between your Ex-Staff and Staff role.

  • purge [limit] {channel (current)}
    Clears a given number of messages.

  • sban [target] {reason}
    Bans a user without messaging them. You can also use the traditional ban method to achieve the same effect.

  • unban [target] {reason}
    Unbans a user.

  • unlock {channel (current)}
    Unlocks the current channel for speaking.

Users

  • pingmod
    Pings the Staff team.

  • staff
    Shows the currently active Staff members.

Infractions

The Appliances have a number of tools to aid in keeping an eye on users and their infractions. For each server, the bot keeps a log of every warning, kick, ban, toss, and note. Here are some ways to manage infractions.

  • clearwarns [target]
    Clears all warnings for a user.

  • delwarn [target] [index] Deletes a warning from a given index.

  • fullinfo [target]
    A combination of info and logs.

  • logs [target]
    Displays logs for a user.

  • note [target] {note}
    Adds a note to a user.

  • unwatch [target] {note}
    Removes a user from the watch list.

  • warn [target] {reason}
    Warns a user.

  • watch [target] {note}
    Puts a user under watch.

Users can also view their own logs with the following command:

  • mylogs
    Lists actions taken on you by Staff.

Tossing

Keep in mind

This feature is currently unimplemented in Refrigerator.

“Tossing” is the act of stripping a user of all of their roles, and replacing it with one single role that forbids speaking in the entire server, save for one channel that gets created on-the-fly for one on one discussions between the offending user and the Staff team.

It’s useful for defusing tense situations and removing problematic users from situations fast.

In addition, the Appliances can archive these sessions for you, whether on Google Drive or saved as a file on the platform itself.

To use this, the Staff and Toss pages in the server configuration must be filled out. To use Google Drive archival, the Archive page must be filled out.

  • archive [user]
    When performed in a Toss channel, will archive the current channel.
    A user can be passed to name the archive after that user.
    When performed elsewhere, will search the archives for a given user.

  • toss [target] {target 2, target 3…}
    Tosses a target, creating a new session.
    If performed in a Toss channel, will add the user(s) to the session.

  • untoss {target/“all”} {target 2, target 3…}
    Untosses a target, restoring their roles and removing them from the session.

  • close
    Archives a session and removes the channel for you.

  • sessions

    Keep in mind

    If this command shows users in a session, yet users are not tossed, this is a known bug. Please contact renavi, or the person hosting the bot.
    To mitigate this issue, always use the untoss command.
    Shows the currently active sessions, and the users in them.

Surveyr

Keep in mind

This feature is currently unimplemented in Refrigerator.

“Surveyr” is the name given to the open source recreation of Pollr within the Appliances. This can track kicks, bans, timeouts, and roles being added or removed. What it tracks can be configured. It (arguably) looks better, and has more functionality whilst being easily configurable. For reference, here are two examples side by side.

To use this, the Surveyr page in the server configuration must be filled out.

  • survey/s
    Posts the 5 recent surveys.

  • reason/r [case ID or “l”]{-case ID for “l”} [reason]
    Records a reason for a survey.

  • censor/c [case ID or “l”]{-case ID for “l”}
    Censors a survey’s name.

  • dump/d [case ID or “l”]{-case ID for “l”}
    Dumps user IDs for surveys. Users can use this too.

Observation

Keep in mind

This feature is currently unimplemented in Refrigerator.

If a Staff channel is configured, the bots will forward users to it who are suspiciously new to Discord (if their account is under 24 hours old). It will then track the user, and forward their first message to the Staff channel. In addition, raidmode can be toggled to force all new users into the Staff channel.

  • raidmode [“on”/“off” (show current state)]
    Toggles Raidmode, a feature to post all new users in the Staff channel.

Noreply

Keep in mind

This feature is currently unimplemented in Refrigerator.

In the event that users on your server do not like being pinged by replies, you can configure Noreply. Noreply tracks each time a user ping replies someone, and pings Staff when their ping count reaches five. Staff are exempt from this.

For new users (within two weeks of joining), the Appliance will ignore the first violation and explain the system.

After this warning (and for each time), the bot will count the infraction in the form of reactions.

The number is how many infractions the user has so far. This count resets daily. In addition, the stop sign can be clicked by the pinged user to forgive the ping, within fifteen seconds.

Puppeteering

If you want to have some fun with the bot, here are the commands to do it with.

  • botnickname {nickname}
    Sets the current nickname. Send command by itself to reset.

  • nickname [target] {nickname}
    Sets a users nickname. Send command by itself to reset.

  • playing [game]
    Sets the bot’s “currently playing” status. Send command by itself to reset.
    Change lasts until next playing rotation.

  • react [channel] [message id] [emoji]
    Puts a reaction on a message.

  • reply [channel] [message id] [text]
    Replies to a message in a given channel with a given text.
    If done by a bot manager, can run any command as the bot.

  • say [text]
    Repeats some text.
    If done by a bot manager, can run any command as the bot.

  • speak [channel] [text]
    Repeats some text in a given channel.
    If done by a bot manager, can run any command as the bot.

  • typing [duration]
    Puts a “Bot is typing…” in chat for a period of time.

Server Configuration

To get the most out of the bot, you’ll need to configure it to work with your server. To edit the configuration, use either of these commands.

  • configs
    Views the current configuration for a server.

  • configs set [category] [setting] [value]
    Sets the current configuration for a server.

The former is recommended for newcomers, while the latter is suggested for those who need to change settings quickly, or if your server’s configuration page is outdated.

Self Hosting and Bot Configuration

Due to both bots being in development with features and code subject to change at any time, no help is currently provided to get it running yourself. This will change with the 1.0 version of the bot.

If you wish to give it a go anyway, you should be able to boot it by simply installing the dependencies and running __init__.py after creating and editing config.py.
